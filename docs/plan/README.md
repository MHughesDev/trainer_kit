# plan/

Lightweight, session-scoped working plans. Use this folder for thinking-in-progress — quick scoping notes, open question lists, spike outlines, or anything you need to write down to think clearly but don't need to keep forever.

> For formal, versioned project plans intended to persist across sessions, use `plans/` instead.

---

## What Belongs Here

- Session scratchpads
- Rough scope drafts ("what are we doing this sprint?")
- Open question trackers
- Quick comparison notes before committing to a decision

## What Does Not Belong Here

- Decisions with long-term architectural impact → `adr/`
- Detailed feature specifications → `specs/`
- Research findings and technology evaluations → `research/`
- Formal project plans with milestones → `plans/`

---

## Conventions

- File names: `kebab-case.md` — include a date prefix if useful: `2024-01-15-auth-spike.md`
- Plans here may be deleted or archived once they are no longer active
- Prefer short files with clear headers over long monolithic notes
